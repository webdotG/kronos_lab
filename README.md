# собрать образ, прислать папку results

- docker с поддержкой gpu nvidia-container-toolkit
- свободная видеокарта - в идеале RTX 4000 Ada (20 ГБ) на ней полетит
- ~15 ГБ места на диск (образ + веса + результаты)
- интернет на этапе сборки (скачает веса модели и зависимости)

проверь, что docker видит gpu:
```
docker run --rm --gpus all nvidia/cuda:12.4.0-base-ubuntu22.04 nvidia-smi
```
если показал твои карты - всё ок. если нет - нужен nvidia-container-toolkit

---

## сборка (один раз, ~10-20 мин)

из этой папки:
```
docker build -t kronos-bt .
```

---

## запуск

ВАЖНО: освободи карту от своей LLM перед запуском (та, что 20 ГБ - RTX 4000 Ada, у тебя это device 1)

```
mkdir -p results
docker run -d --name kronos-bt \
  --gpus '"device=1"' \
  -v "$(pwd)/results:/app/kronos_lab/data" \
  kronos-bt --n_points 1200 --step 24 --n_paths 500 --chunk 128
```

что это значит:
- `--gpus '"device=1"'` - использовать карту 1 (RTX 4000 Ada 20 ГБ). если свободна другая - поменяй номер
- `-v .../results:...` - результаты пишутся в папку results на твоём диске (переживут перезапуск)
- `-d` - в фоне. прогон долгий (~1.5-2.5 суток), пусть крутится
- `--chunk 128` - размер батча. если будет ошибка памяти - скрипт сам уменьшит, ничего не делай

первый запуск сам скачает историю BTC с Bybit (пара минут), потом пойдёт счёт

## следить за прогрессом

```
docker logs -f kronos-bt
```
увидишь строки вида:
```
[06-08 15:30:01] [42/1200] 2025-03-14 11:00:00 up=98.0% ход=4.1% реал=-1.2% chunk=128 ETA~38.5ч
```
ETA покажет, сколько осталось. можно закрыть терминал, прогон в фоне продолжится

если прервалось (перезагрузка, упало) - просто запусти ту же команду docker run снова
(сначала `docker rm kronos-bt`), он ПРОДОЛЖИТ с места, не с нуля - резюм встроен

## когда закончит

в логах будет `готово. посчитано в этом запуске: ...`
пришли Кириллу всю папку `results/` - там:
- `backtest_journal.csv` - главный файл, метрики по каждой точке
- `backtest_ensembles/` - сырые ансамбли и реальные пути (тяжёлая папка, но нужна для анализа)

можно заархивировать: `tar czf results.tar.gz results/` и прислать архив
